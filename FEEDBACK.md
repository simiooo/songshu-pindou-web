## 🔴 P0 - 紧急修复

### 1. 图片转换效果差

**问题描述:**
- 边界线条不清晰
- 出现乱码/噪点
- 整体转换质量不佳

**影响范围:**
- 桌面端和移动端均有此问题
- 手机端效果尤其差

**根因分析:**
- 当前像素化算法（K-Means + 色号匹配）在处理复杂图像时效果不理想
- 颜色量化过程丢失细节
- 边界检测和保留不够精准

**解决方案:**
1. **更换算法**: 尝试以下替代方案
   - 基于边缘检测的算法（保留轮廓）
   - 超像素分割算法（SLIC）
   - 深度学习方案（如需）
2. **增加预处理**: 锐化、对比度增强
3. **参数调优**: 调整 K-Means 聚类参数
4. **后期处理**: 边界平滑、去噪

**解决方案实现:**
1. **双线性插值采样**: 替换原有的最近邻采样，获得更平滑的颜色过渡
2. **边缘检测与保留**: 使用 Sobel 算子检测边缘，在边缘区域使用最近邻采样保持锐利
3. **图像预处理**: 添加锐化和对比度增强
4. **有序抖动算法**: 使用 Bayer 矩阵的有序抖动替代 Floyd-Steinberg 抖动，减少噪点

**修改文件:**
- `src/utils/pixelation.ts`: 完整重写像素化算法

**验收标准:**
- [x] 转换后的图案边界清晰
- [x] 无明显噪点/乱码
- [x] 整体视觉效果可接受

**状态:** ✅ 已完成 (2026-03-21)
